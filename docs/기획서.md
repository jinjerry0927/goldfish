# 🐠 GoldFish 기획서

> 금융 분석 원(圓) → **금붕어(GoldFish)**
> 내 투자·거래 데이터를 어항 들여다보듯 분석해주는 도구

---

## 1. 한 줄 정의

> **내 투자·거래 데이터를 넣으면, 통계 분석 + 금융 특화 진단 + AI 자연어 코칭을 담은 리포트(텍스트+차트)를 자동 생성하는 Python 라이브러리/CLI**
>
> 데이터 소스: CSV / 토스증권 OpenAPI (read-only)

---

## 2. 왜 만드는가 (동기)

- **개인 목표**: 사람들이 `pip install` 한 줄로 가져다 쓰는 오픈소스를 만들고 싶다. AI + 데이터 분석을 제대로 공부하고 싶다.
- **시장 빈틈**:
  - 기존 프로파일링 도구(`ydata-profiling` 등)는 **범용**이라 금융 특화 점검(포트폴리오 집중도, 매매 습관, 수익률·변동성)이 없다.
  - 거기에 **"AI가 사람 말로 진단·코칭"** 해주는 도구는 거의 없다 → GoldFish의 차별점.
- **수요**: 투자/거래 데이터를 받으면 누구나 같은 점검을 반복한다. 자동화 가치가 크다.

---

## 3. 타겟 사용자

1. 자기 투자 데이터를 분석하고 싶은 **개인 투자자**
2. 거래 데이터를 빠르게 진단해야 하는 **데이터 분석가 / 핀테크 개발자**
3. 금융 데이터 분석을 학습하려는 **학생·주니어 개발자**

---

## 4. 핵심 기능

### 입력 (Loaders)
- `load_csv(path)` — CSV 파일
- `load_toss(...)` — 토스증권 조회 API (read-only, 선택 모듈)

### 분석 (Analyzers) — *AI 없이 순수 pandas로 동작*
- **기본 프로파일링**: 결측치 / 데이터 타입 / 분포 / 이상치 / 상관관계
- **금융 특화 진단**:
  - 포트폴리오 집중도·분산 (종목별 비중, 섹터 편중)
  - 매매 패턴 (과매매 빈도, 손절·익절 습관)
  - 수익률·변동성 통계

### AI 요약 (AI Layer) — *분석 결과 위에 얹는 선택 계층*
- LLM(**Gemini API 무료 티어**)이 분석 결과를 **자연어로 해설·경고·코칭**
- 예: "⚠️ 23시 이후 고액 매매 비중이 비정상적으로 높습니다. 충동 거래 가능성을 점검하세요."
- **키 없으면 통계만**: AI는 켜고 끄는 선택 기능. 설치/서버 불필요(클라우드 무료 티어), 유일한 준비물은 무료 Gemini 키 1개.

### 출력 (Report)
- 차트(matplotlib) + 텍스트 → **HTML 리포트**
- `report.summary` 로 자연어 요약만 따로 추출 가능

---

## 5. 범위 (Scope)

### ✅ In scope
- CSV / 토스 조회 데이터 입력
- 기본 + 금융 특화 분석
- AI 자연어 진단·코칭
- 차트 + 텍스트 HTML 리포트

### ❌ Out of scope
- **자동 매매 / 주문 실행** (위험 — read-only 원칙)
- 실시간 트레이딩 봇
- **투자 추천** ("이거 사라") — 법적 리스크. "진단·설명"까지만.
- 웹/앱 UI (라이브러리 + CLI 형태 유지)

---

## 6. 핵심 안전 원칙

1. 토스 API는 **조회(read)만** — 주문(write) 절대 금지
2. API Key / Secret Key는 **`.env`에만** 저장, 깃 커밋 금지 (`.gitignore` 필수)
3. **"추천"이 아니라 "진단·설명"** — 투자자문이 아님 (면책 문구 README 명시)
4. 실제 거래/고객 데이터는 repo에 포함하지 않음 (샘플은 합성 데이터)
5. 이름·로고에 "토스" 공식 표기 금지 ("비공식 연동" 명시)

## 7. 기술 스택

| 영역 | 선택 |
|---|---|
| 언어 | Python 3.10+ |
| 분석 | pandas, numpy |
| 차트 | matplotlib (→ 추후 plotly) |
| AI | Gemini API (`google-genai`, 무료 티어) — 선택 계층 |
| CLI | typer |
| 설정 | python-dotenv |
| 테스트 | pytest |
| 배포 | pyproject.toml + PyPI |

## 8. 폴더 구조

```
goldfish/
├── goldfish/
│   ├── __init__.py
│   ├── loaders/          # 데이터 입력
│   │   ├── csv.py
│   │   └── toss.py       # 토스 조회 API → DataFrame (선택)
│   ├── analyzers/        # 분석 (AI 없음, 순수 pandas)
│   │   ├── basic.py      # 결측/타입/분포/상관
│   │   └── finance.py    # 포트폴리오/매매패턴/수익률
│   ├── ai/               # LLM 요약
│   │   └── summarize.py
│   ├── report/           # 출력
│   │   ├── charts.py
│   │   └── html.py
│   └── cli.py
├── tests/
├── examples/             # 샘플 합성 데이터 + 사용 예시
├── docs/
│   └── 기획서.md
├── .env.example
├── .gitignore
├── README.md
└── pyproject.toml
```

> 설계 원칙: **`analyzers`는 AI/토스 없이도 완전히 동작**해야 한다. AI와 토스는 위에 얹는 선택 계층.
